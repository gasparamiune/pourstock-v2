

## Diagnosis

The live app at `www.pourstock.com` crashes on load with:

```
Error: supabaseUrl is required.
```

**Root cause**: The `.env` file — which contains `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` — is missing from the project. Without these variables, the Supabase client cannot initialize, and the entire app fails before rendering anything.

This file is auto-managed by Lovable Cloud and was likely removed during the recent repository cleanup (the previous message deleted `src/App.css` and edited root files).

## Fix

**No code changes are needed.** The `.env` file is auto-generated by Lovable Cloud with the correct backend credentials. The system should regenerate it automatically on the next build/deploy cycle.

### If it does not regenerate automatically:

1. Trigger a re-publish of the app (click **Publish → Update** in the Lovable editor) — this should restore the `.env` with the correct values.
2. If the file still doesn't appear, the Cloud connection may need to be re-linked — navigate to **Settings → Cloud** to verify the backend connection is active.

### What NOT to do:

- Do **not** manually create the `.env` file — it is auto-generated and would be overwritten.
- Do **not** hardcode the Supabase URL or key into `client.ts` — that file is also auto-generated.

## Verification

After the `.env` is restored and the app is re-published:
- The runtime error will disappear
- The app will load normally at `www.pourstock.com`
- All backend features (auth, data, edge functions) will work
