# PourStock Monetization Model

## Subscription Tiers

| Plan | Target | Description |
|------|--------|-------------|
| **Starter** | Small hotels (≤30 rooms) | Core modules: Inventory, Table Plan, Purchase Orders |
| **Professional** | Mid-size hotels (31–100 rooms) | All modules + advanced reporting + AI parsing |
| **Enterprise** | Hotel groups / chains | Multi-property dashboards, centralized config, priority support |

## Revenue Streams

1. **Monthly SaaS subscription** — per-hotel, tiered by plan
2. **AI usage** — included in Professional+; overage billing for high-volume parsing
3. **Premium integrations** — PMS/POS connectors (Mews, Opera) as add-on modules
4. **Onboarding & setup** — optional professional services for initial configuration

## Pricing Philosophy

- **No per-user fees** — hospitality teams rotate staff frequently; per-user pricing creates friction
- **Per-hotel pricing** — aligns with how hotel groups budget (per property)
- **Module-based upsell** — hotels start with what they need, expand over time

## Current Status

The platform operates on a **Starter plan** default for all hotels. Billing infrastructure is not yet active — all features are available during the pilot phase at Sønderborg Strand Hotel.

## Deferred Considerations

- Stripe integration for automated billing (deferred until multi-hotel onboarding)
- Usage-based AI metering dashboard
- Annual vs. monthly discount tiers
